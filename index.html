Technical Documentation & Quick Start

Technische Dokumentation & Schnellstart

Learn how to easily export your Siemens TIA Portal blocks into SCL/XML formats to feed our offline compiler.

Erfahren Sie, wie Sie Ihre Siemens TIA Portal Bausteine ganz einfach in die Formate SCL und XML exportieren können.

1 Method A: Version Control Interface (VCI) [Recommended] Methode A: Version Control Interface (VCI) [Empfohlen]

The TIA Portal Version Control Interface is the cleanest way to perform bulk XML and SCL exports without installing Openness. Follow these steps:

Das Version Control Interface im TIA Portal ist der einfachste Weg, XML- und SCL-Massenexporte ohne TIA Openness durchzuführen:

  • In TIA Portal, open your project and navigate to the project tree. Scroll down and open Version control interface.
  • Öffnen Sie Ihr TIA Portal-Projekt. Navigieren Sie im Projektbaum nach unten und öffnen Sie das Version control interface (VCI).
  • Configure a new Workspace and point its target path to an empty directory on your local machine (e.g. C:\TIA_Exports).
  • Erstellen Sie einen neuen Arbeitsbereich (Workspace) und verknüpfen Sie diesen mit einem leeren Ordner auf Ihrer Festplatte (z. B. C:\SPS_Exporte).
  • Drag and drop your target folders (e.g., Program blocks and PLC data types) from the TIA tree directly into the VCI Workspace pane.
  • Ziehen Sie Ihre gewünschten Ordner (z. B. Programmbausteine und SPS-Datentypen) per Drag-and-Drop direkt in das VCI-Arbeitsbereichsfenster.
  • TIA Portal will automatically serialize and write all blocks as compliant SCL, DB, and XML files into your directory. Select this folder in Logic Analyzer Pro.
  • Das TIA Portal generiert automatisch alle Bausteine als SCL-, DB- und XML-Dateien in Ihrem Zielordner. Wählen Sie diesen Ordner in Logic Analyzer Pro aus.

2 Method B: Copy-As-Text (Manual SCL Generation) Methode B: Kopieren als Text (Manuelle SCL-Generierung)

If you only need to audit a few isolated blocks, you can generate SCL files manually:

Wenn Sie nur wenige, einzelne Bausteine analysieren möchten, können Sie SCL-Dateien manuell erstellen:

  • In TIA Portal, double-click to open your target SCL Function Block (FB) or Function (FC).
  • Öffnen Sie im TIA Portal den gewünschten SCL-Funktionsbaustein (FB) oder die Funktion (FC) per Doppelklick.
  • Press Ctrl + A to select the entire block body, right-click, and select Copy (or Copy as Text).
  • Drücken Sie Strg + A, um den gesamten SCL-Code zu markieren. Klicken Sie mit der rechten Maustaste und wählen Sie Kopieren.
  • Open a text editor (e.g. Notepad), paste the copied code, and save the file with a .scl extension (e.g., Motor_Ctrl.scl). Place it inside your scanning folder.
  • Öffnen Sie einen Texteditor (z. B. Editor), fügen Sie den Code ein und speichern Sie die Datei mit der Endung .scl (z. B. Motor_Ctrl.scl) in Ihrem Analyseordner.

Frequently Asked Questions

Clear answers to common questions about security, VM licensing, and local execution.

Q: Do my proprietary industrial PLC codes leave my computer? F: Werden meine geschützten SPS-Quellcodes ins Internet übertragen?

A: No. Security is our highest priority. The parsing, NetworkX topological calculations, and rendering happen 100% locally and offline on your computer. If you use the local AI model (Ollama), no internet connection is used. Only if you actively choose a cloud model (OpenAI) is the specific selected code block transmitted to their secure API.

A: Nein. Sicherheit steht bei uns an erster Stelle. Das Einlesen, die NetworkX-Berechnungen und die Darstellung erfolgen zu 100% lokal und offline auf Ihrem Computer. Wenn Sie das lokale KI-Modell (Ollama) nutzen, wird keinerlei Internetverbindung aufgebaut. Nur wenn Sie aktiv ein Cloud-Modell (OpenAI) auswählen, wird der gewählte Codeblock zur Analyse übertragen.

Q: Can I run this software inside a Virtual Machine (VM)? F: Kann ich die Software in einer virtuellen Maschine (VM) ausführen?

A: Yes. Unlike many licensing systems that block VMs, VM execution is fully supported out-of-the-box on our TEAM and SITE tiers. You can safely activate and run the software inside your engineering VMs (VMware, VirtualBox, Hyper-V) used on-site.

A: Ja. Im Gegensatz zu vielen Lizenzsystemen, die VMs blockieren, wird die Ausführung in virtuellen Maschinen (VMware, VirtualBox, Hyper-V) auf unseren TEAM- und SITE-Stufen standardmäßig unterstützt.

Q: How do licensing workstation seats work? F: Wie funktionieren die Workstation-Aktivierungen (Seats)?

A: A single PRO license allows activation on 1 unique physical machine/VM. A TEAM license allows activation on up to 5 unique physical devices or VMs concurrently. If you need to migrate your license to a new machine, you can deactivate the current key in the app settings or contact our support.

A: Eine einzelne PRO-Lizenz ermöglicht die Aktivierung auf 1 physischen PC oder 1 VM. Eine TEAM-Lizenz erlaubt bis zu 5 zeitgleiche Aktivierungen. Wenn Sie Ihre Lizenz auf einen neuen Rechner übertragen möchten, können Sie die Lizenz in den Einstellungen deaktivieren oder unseren Support kontaktieren.

Q: How do I report bugs or request new features? F: Wie kann ich Fehler melden oder neue Funktionen anfragen?

A: Please send all technical bug reports, diagnostic logs (logic_analyzer_diagnostics.log), or feature requests directly to our support desk: support@logic-analyzer-pro.de. Our engineering team usually responds within 24 business hours.

A: Bitte senden Sie alle technischen Fehlerberichte, Diagnoseprotokolle (logic_analyzer_diagnostics.log) oder und Funktionsanfragen direkt an: support@logic-analyzer-pro.de. Unser Team antwortet in der Regel innerhalb von 24 Stunden.
